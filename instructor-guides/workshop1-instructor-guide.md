# Instructor Guide: Workshop 1

Workshop: Foundations & Capabilities  
Length: 60 minutes  
Presentation target: 15-18 minutes  
Main tools: ChatGPT Edu, Drexel AI Tools page  
Participant assumption: limited prior use, uneven confidence

## Goals

Participants should leave able to:

1. Explain what a large language model does in practical terms.
2. Choose Drexel-approved tools based on task and data classification.
3. Use ChatGPT Edu with GPT-5.5 when available.
4. Write and revise one useful prompt using Persona, Task, Context, Format.

## Instructor Prep

- Open ChatGPT Edu and confirm you can sign in with Drexel credentials.
- Confirm whether GPT-5.5 appears in the model picker. If it does not, say: "Use the current default or latest GPT model in the Drexel workspace."
- Open the Drexel AI Tools page in a tab.
- Have one public paragraph ready to summarize. Use a public DSPH web page, public report, or made-up content.
- Have a back-up plan if someone cannot sign in: they can pair with a neighbor or use Copilot Chat with Enterprise Data Protection.

## Slide Plan and Speaking Notes

Use 6 slides. Do not spend more than 3 minutes on any slide.

### Slide 1: What We Are Doing Today

Claim: The goal is not to learn every AI feature. The goal is to get one safe, useful workflow started.

Say:

- "We have one hour. I want you to leave having used ChatGPT Edu for a real task, not just heard about it."
- "We will focus on low-risk examples first and use Drexel's data rules as the boundary."

### Slide 2: A Practical Mental Model

Claim: AI is useful for drafting, summarizing, rewriting, explaining, and reformatting.

Say:

- "The model predicts likely responses from patterns in text. It can produce good drafts quickly."
- "It can also be wrong, generic, or overconfident. Your judgment is the quality control step."

Reading to mention: `Judgment is the skill that matters most in the AI era – Jim Grey on software man.pdf`

### Slide 3: Tool Choice Starts With Data

Claim: Pick the tool based on where the work lives and what data are involved.

Say:

- "For Drexel work, start with Drexel-provided tools."
- "Consumer tools are Low Risk Data only unless Drexel has separately approved the use case."
- "When data are protected, identifiable, contractual, student-related, HR-related, or research-sensitive, stop and check."

Demo: Open the Drexel AI Tools page and point to ChatGPT Edu, Copilot Chat, Microsoft 365 Copilot, and Zoom AI Companion.

### Slide 4: Use ChatGPT Edu for This Session

Claim: ChatGPT Edu is appropriate for drafting, summarizing, file-based analysis, and reusable assistants when the data and use case are allowed.

Say:

- "Use GPT-5.5 if it appears. If it does not, use the current default or latest GPT model in the Drexel workspace."
- "Do not paste sensitive data for practice. Use public, synthetic, or low-risk examples."

### Slide 5: The Prompt Recipe

Claim: PTCF gives participants a repeatable way to get better output.

Say:

- "Persona: who should it act as?"
- "Task: what should it do?"
- "Context: what does it need to know?"
- "Format: what should the answer look like?"

Demo prompt:

> You are a research coordinator. Draft a short project status update for a monthly team check-in. Context: recruitment is complete, data cleaning is behind schedule, and the IRB renewal is due next month. Format: Done, In Progress, Needs Attention. Keep it under 150 words.

### Slide 6: What Good Use Looks Like

Claim: The workflow is prompt, inspect, revise, verify.

Say:

- "The first answer is not the product. It is the starting point."
- "Check facts, numbers, citations, tone, and anything that affects a real decision."
- "Save prompts that worked. We will build reusable recipes in Workshop 2."

## Activity Facilitation Notes

Activity 1 should build confidence. Do not over-explain model architecture. Walk the room and help people get one successful output.

Activity 2 should make the improvement visible. Ask participants to compare the bare prompt with the PTCF prompt. The goal is for them to see that specificity changes the output.

If someone wants to use sensitive work content, redirect them:

> Use a synthetic version with the same structure. Replace names, dates, dollar amounts, participant details, and anything identifiable.

## Common Questions

**Can I use this for student or research data?**  
Only when the tool and use case are approved for that data classification. For research data, also check the IRB and any data use agreement.

**Can I trust citations?**  
No. Treat citations as unverified until checked against the source.

**Should I use ChatGPT Edu or Copilot?**  
Use ChatGPT Edu for open-ended drafting, summarizing, files, and custom GPTs. Use Microsoft 365 Copilot when the work is already in Word, Outlook, Excel, PowerPoint, or Teams.
